# 웹사이트의 기술적 SEO를 개선하기 위해 고려해야 할 6가지

현재 회사에 입사했을 때, 제가 처음으로 개선한 것 중 하나는 SEO였습니다.
당시에는 이 작업이 얼마나 길고 힘든 일일지 전혀 상상하지 못했고, 제 팀 리더도 마찬가지였을 것이라고 확신합니다.

**기술적 SEO**는 검색 엔진이 웹사이트를 이해하고 인덱싱할 수 있도록 돕는 백엔드 최적화를 의미합니다.
이는 검색 엔진 결과 페이지(SERP)에서 웹사이트의 가시성과 순위를 **개선**하는 데 중요한 요소입니다.
이 작업은 사이트 속도와 성능을 개선하고, 모바일 친화적으로 만들며,
검색 엔진 봇이 웹사이트의 콘텐츠를 쉽게 크롤링하고 인덱싱할 수 있도록 보장하는 다양한 기술을 통해 달성됩니다.

## 1. 사이트 구조 및 아키텍처

사이트 구조 및 아키텍처. 기술적 SEO 감사에서는 검색 엔진 크롤러가 웹사이트의 페이지를 쉽게 탐색하고 인덱싱할 수 있는지 확인하기 위해
웹사이트의 구조와 아키텍처를 검토합니다.
저의 경우 URL 구조에서 불필요한 URL 깊이를 제거하여 SEO 통계를 개선했습니다.
예를 들어, 제품 페이지의 URL 경로는 `/product/detail/{productId}`였습니다.
`/product/detail` 페이지는 존재하지 않았고 404 페이지였습니다.
저는 URL 깊이인 `/detail`을 제거하고, 제품 페이지의 URL 경로를 단순히 `/product/{productId}`로 설정했습니다.
또한, 콘텐츠를 논리적인 카테고리와 하위 카테고리로 구성하고,
사용자와 검색 엔진이 페이지의 계층 구조를 이해할 수 있도록 빵부스러기 내비게이션을 사용하는 것도 포함됩니다.

## 2. HTML 태그의 적절한 사용

웹사이트에서 적절한 태그와 제목을 사용하는 것이 중요합니다. 여기에는 브라우저 탭에 표시되는 제목 태그와
검색 결과에 표시되는 메타 설명이 포함됩니다.
이 태그들은 페이지의 콘텐츠와 관련이 있어야 하며, 중복 콘텐츠가 없어야 합니다.
또한, 웹사이트에 고품질의 고유한 콘텐츠를 포함하는 것도 중요합니다. 여기에는 잘 작성되고 유익한 기사뿐만 아니라,
콘텐츠가 웹사이트의 타겟 청중과 관련이 있어야 한다는 것이 포함됩니다.

## 3. 페이지 속도와 성능

웹사이트의 속도와 성능은 사용자 경험과 검색 엔진 순위에 중요한 요소입니다.
기술적 SEO 감사에서는 페이지 로드 시간, 서버 응답 시간, 캐싱 및 기타 성능 향상 기법을 검토하여
웹사이트가 빠르고 효율적인지 확인합니다.
웹사이트 속도를 개선하는 방법으로는 이미지 최적화, 코드 축소, 콘텐츠 전송 네트워크(CDN) 사용 등이 있습니다.

## 4. 모바일 친화성

점점 더 많은 사용자가 모바일 기기를 통해 웹을 이용하고 있으므로 웹사이트가 모바일에 최적화되어 있어야 합니다.
기술적 SEO 감사에서는 모바일 기기에서 웹사이트의 디자인과 레이아웃, 반응형 디자인 사용 여부,
클릭 투 콜 버튼 및 위치 맵과 같은 모바일 친화적 기능 구현을 검토합니다.
Google에는 웹사이트가 모바일 친화적인지 여부를 확인할 수 있는 Mobile-Friendly Test라는 도구가 있습니다.

## 5. 보안 및 개인정보 보호

웹사이트와 사용자의 보안 및 개인정보를 보장하는 것은 신뢰와 신뢰성을 구축하는 데 필수적입니다.
이것은 특히 전자상거래 웹사이트의 경우 중요한데, 신용카드 번호나 개인 데이터를 보호하는 데 도움이 되기 때문입니다.
URL의 시작 부분에 "http" 대신 "https"가 있는지 여부로 보안 연결을 확인할 수 있습니다.

## 6. 모니터링

SEO 작업을 몇 개월 동안 진행한 후 한 가지를 깨달았습니다. SEO는 Search Engine Optimization(검색 엔진 최적화)이 아니라 Suffering Endless Optimization(끝없는 최적화 고통)의 약자라는 것입니다.
웹사이트의 기술적 SEO를 정기적으로 모니터링하는 것이 중요합니다.
여기에는 끊어진 링크를 확인하고, 웹사이트가 모바일 친화적인지 확인하며,
웹사이트의 속도를 모니터링하는 것이 포함됩니다. 웹사이트의 기술적 측면을 정기적으로 모니터링함으로써
검색 엔진 가시성에 부정적인 영향을 미칠 수 있는 문제를 식별하고 수정할 수 있습니다.
웹사이트를 완전히 통제하지 못하는 경우, SEO 통계를 정기적으로 모니터링하는 것이 매우 중요합니다.
